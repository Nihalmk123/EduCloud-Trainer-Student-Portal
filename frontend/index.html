Trainer uploads file

Trainer logs in to the Linux‑backed portal, selects course, and uploads materials.

  • • Linux user & home dir ensure separation.
  • • File lands in a staging folder on EC2.
02

Platform secures & stores

A background process pushes files to S3, tags them, and writes metadata to RDS.

  • • S3 versioning + encryption at rest.
  • • Trainer, course, and expiry stored in RDS.
03

Student downloads safely

Student sees only assigned files; downloads use short‑lived CloudFront links.

  • • No direct S3 exposure to the client.
  • • All access logged for audits.
